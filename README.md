# ai-system

Personal AI operating system — a collection of markdown files, prompts, and logs that power automated AI agents via Claude Code cloud scheduled tasks.

## Purpose

This repo is the backbone of a personal AI system. Cloud scheduled tasks (running on Anthropic's infrastructure) clone this repo on every run, read the relevant files, perform their task, and commit results back. The owner interacts with the system by:

1. Editing files in this repo (locally via Obsidian, GitHub mobile app, or Claude Code)
2. Reading outputs generated by scheduled tasks (coaching messages, digests, reviews)
3. Chatting with Claude.ai for deeper conversations (Claude has memory of this system)

### How scheduled tasks work

Each cloud scheduled task:
1. Clones this repo fresh at the scheduled time
2. Reads the files specified in its prompt
3. Generates output (coaching message, digest, review)
4. Writes output to the appropriate folder
5. Commits and pushes with a descriptive message
6. Can also access Gmail and Google Calendar via connectors

Tasks run on Anthropic's servers — no local machine needs to be running.

## File conventions

### For AI agents reading this repo

- **System prompt files** (`*-system-prompt.md`): Read these first. They contain the persona, rules, knowledge base, and constraints for that domain. Follow the rules defined in them.
- **Log files** (`training-log.md`): Human-edited, append-only (newest entries at top). Never delete existing entries. Only add new entries if the human explicitly asks.
- **Program/plan files** (`program-current.md`, `goals-current.md`): Agent-updatable. When modifying, update the "Last updated" date and explain what changed in the "Notes from coach" section.
- **Output folders** (`daily-coaching/`, `weekly-reviews/`, `newsletter-digests/`): Agent-written. Use the filename pattern shown. One file per period. Overwrite if re-running for the same date.
- **Topics of interest** (`topics-of-interest.md`): Reference when filtering or prioritizing content. Respect the priority tiers.

### Commit messages

Use this format for automated commits:
- Daily coaching: `coach: YYYY-MM-DD`
- Weekly review: `review: week YYYY-WW`
- Program update: `program: [brief description]`
- Newsletter digest: `digest: week YYYY-WW`
- Reflection: `reflect: week YYYY-WW`

### Markdown formatting

- Use ATX headings (`#`, `##`, `###`)
- Tables for structured data (programs, targets)
- Bold for exercise names in training logs
- ISO dates everywhere: `YYYY-MM-DD`
- Newest entries at the top of log files

## Scheduled tasks

### Daily fitness coach
- **Schedule:** Every day at 7:00 AM Europe/Berlin
- **Model:** Sonnet 4.6
- **Reads:** `fitness/coach-system-prompt.md`, `fitness/training-log.md`, `fitness/program-current.md`
- **Writes:** `fitness/daily-coaching/YYYY-MM-DD.md`
- **Behavior:** Generate today's coaching message based on the program schedule and recent log entries. Adjust for recovery signals. Nudge if log hasn't been updated in 2+ days.

### Weekly training review
- **Schedule:** Every Sunday at 8:00 PM Europe/Berlin
- **Model:** Sonnet 4.6
- **Reads:** `fitness/training-log.md`, `fitness/program-current.md`, `fitness/coach-system-prompt.md`
- **Writes:** `fitness/weekly-reviews/week-YYYY-WW.md`, may update `fitness/program-current.md`
- **Behavior:** Analyze the past 7 days of training. Report on volume, adherence, recovery patterns. Suggest program adjustments. Update program-current.md if changes are needed.

### Newsletter digest (planned)
- **Schedule:** Every Monday at 8:00 AM Europe/Berlin
- **Connector:** Gmail (news.fabian@gmail.com)
- **Reads:** `second-brain/topics-of-interest.md`, Gmail inbox
- **Writes:** `second-brain/newsletter-digests/digest-YYYY-WW.md`
- **Behavior:** Read unread newsletter emails from the past 7 days. Extract top insights organized by topic (not by newsletter). Filter by topics-of-interest priorities. Max 500 words.

## How to interact with this system

### Log a workout
Edit `fitness/training-log.md` — add a new entry at the top following the template format. Can also be done via Obsidian (Mac), GitHub mobile, or by telling Claude via Dispatch.

### Save a research note
Add a markdown file to `second-brain/inbox/` with a descriptive filename.

### Change training program
Edit `fitness/program-current.md` directly, or ask Claude in chat to suggest changes.

### Ask for advice
Open Claude.ai, Claude Code, or use Dispatch from your phone. Claude has memory of this system and can reference files when you mention them.
